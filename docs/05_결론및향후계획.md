# 5. 결론 및 향후 계획

## 5.1 결론

본 보고서는 방산 UAV 임무 시스템을 대상으로, 공격과 방어를 하나의 폐루프로 묶은 AI 에이전트 체계를 설계하고 구현했다. 공세 측 `fried-pollack-ai`(OCO)와 방어 측 `pollack-ai`(DCO-IDM)를 별도 저장소·별도 런타임으로 분리하고 두 에이전트를 코드가 아니라 UAV*_CL 텔레메트리 스키마 하나로만 잇는 one-way 브릿지 위에서 교전시켰다. 설계의 출발점은 모델 성능이 아니라 조직과 교리였고, 그 결과 "AI는 판단을 돕고 권한은 코드와 정책이 쥔다"는 원칙이 저장소 경계·그래프 순서·승인 게이트·로그 스키마로 구현되었다.

이 세 장은 하나의 입력 연쇄로 이어진다. 2장은 `uav-sim-env` 자산을 xT-STRIDE로 위협 모델링하고 MITRE ATT&CK for UAV(15전술·116기법)로 정규화한 뒤, 계층·효과·취약 전제·재현성 네 축에서 직교하는 세 대표 시나리오(A 제어권 탈취·B 영상 변조·C 데이터 유출)를 확정했다. 3장은 그 공격을 입력으로 받아 탐지룰·Golden Fixture·Runbook·CACAO Playbook을 전술판 기준으로 사전 연계한 사전 정의 계약, D3FEND 방어 매핑, 다층 방어, 그리고 공격 이후의 임무 지속성을 판정하는 Mission Assurance에 이르는 방어 아키텍처를 수립하였다. 이어 4장은 그 두 에이전트를 LangGraph 상태머신으로 구현하고 RoE 게이트·단발 HITL 토큰·out-of-band 오라클로 권한을 모델 밖에 두었으며, RAG는 조사 보강에만 활용하고 판정은 결정론 엔진이 담당하도록 하였다. 구현 근거는 모듈·원자 액션·기법 커버리지·벤치마크·테스트 수와 컴포넌트별 LoC로 제시하고, 문서 수치는 CI가 committed 아티팩트에서 재파생해 검증한다.

핵심 기여는 네 가지다.

- **폐루프 적대적 검증**: 방어를 문서상 가정이 아니라 실제 공격 행위로 검증한다. 레드팀 에이전트가 방어 배포 이전에 공백을 먼저 찾는 전방 방어(defend forward) 구조다.
- **결정론 권한 경계 + HITL**: 교전권한·심각도 하한·정탐 판정·대응 실행·룰 배포는 결정론 코드가 쥐고, 실행에는 인간 승인이 필요하며, LLM은 계획 보조·조사 요약·근거 정리에 한정된다. 군 임무 시스템에 적용 가능한 안전 경계다.
- **정직한 갭 계량**: SOC 탐지 커버리지 KPI 대시보드가 룰 커버 폭(전술 93%·기법 80%)과 얕은 배포 성숙도(depth 3.6%)를 분리해 드러내고, 미커버·계획 기법을 그대로 배포 우선순위로 환산한다. 이로써 측정이 개선 행동으로 연결된다.
- **재현성과 무결성**: 외부 의존 없이 표준 라이브러리만으로 재현되는 Tier 0 경로와, 재현 불가능한 능력을 능력으로 세지 않는 CI 정직성 가드를 함께 둔다.

## 5.2 한계

본 체계는 설계·로직·테스트를 완비하였으며 Tier 0 경로에서 외부 의존 없이 종단 재현된다. 다만 실 운영 환경으로 하드닝할 때 남은 항목을 정직하게 기술한다.

- **탐지 심화 배포**: 룰 커버리지 폭은 넓고(전술 93.3%·기법 80%·165룰), 정밀 탐지 깊이를 갖춘 것은 네이티브 4 + 프록시 6이다. `pollack-ai`에 직접 배포된 Analytic Rule은 `S1`(GpsSpoofDetector)이며, 미커버 15·계획 7 기법과 C 시나리오 골든 픽스처가 순차 심화·편입 대상이다.
- **근사(proxy) 탐지**: `S88`(적대적 패치)은 온보드 AI 추론 전용 텔레메트리 부재로 운영 감사 이벤트를 근사 신호로 사용하며, 센서·모델 텔레메트리 계측으로 실 신호 대체가 남는다.
- **평가 하네스**: PyRIT 연동은 통합 지점(스켈레톤)까지 마련했고, 실 주입 실행은 외부 레드팀 lane과의 연결이 남는다.
- **Azure 실 구독 배선(약 10%)**: LLM 프로덕션 교체(Azure OpenAI), 주기 워커 스케줄러(AKS CronJob), Sentinel/Azure Monitor 라이브 경로는 배포 IaC(`pollack-infra`)와 로직·계약·테스트까지 준비돼 있고, 실 구독 apply만 남는다.

## 5.3 향후 계획

앞서 짚은 한계를 다음 순서로 닫는다.

- **Azure 실 배포**: 배포 IaC는 `pollack-infra`(구독 스코프 Bicep)로 완비돼 있다. 실 구독에 apply해 AKS(레드·심)·ACR·Azure OpenAI·Log Analytics/Sentinel을 구축하고, LLM 실 교체·AKS CronJob 워커 스케줄러·Sentinel 라이브 경로를 연결해 위 "약 10%"를 닫는다.
- **탐지 배포 확대**: `S24`·`S88` 등 파일명 계약 상태의 탐지룰을 실 배포 Analytics Rule로 승격하고, 온보드 AI 추론 텔레메트리를 계측해 `S88` 근사 룰을 실 신호로 대체한다.
- **골든 픽스처 편입**: C 시나리오(`S35`·`S92`·`S93`·`S96`) 골든 픽스처를 추가해 A·B와 같은 회귀 기준으로 잠근다.
- **커버리지 깊이 개선**: 미커버 15기법·계획 7기법을 실 배포 Analytic Rule로 승격하고, 룰 성숙도(깊이)를 네이티브 탐지로 끌어올린다. 재교전(PoV 페어)으로 개선을 계량한다.
- **평가 자동화**: PyRIT 완전 연동과 지속 교전 자동화로, 공격·방어 재교전을 CI 주기에 편입한다.
- **운영 튜닝**: `RuleUpdateAgent`의 워치리스트 임계치·예외를 실 교전 오탐 분포에 맞춰 조정하고, least-change 보정 주기를 확립한다.
- **대응 지연 상시 계측**: 실 배포 후 티켓 대응 지연(MTTD/MTTC)을 상시 계측해 회귀 기준선으로 삼고, 시나리오별 목표 지연을 관리한다.
- **KPI 대시보드 실시간 추적**: SOC 탐지 커버리지 KPI 대시보드를 교전 결과에 따라 실시간 갱신하고, 전술·기법 커버리지 드리프트를 추적해 배포 우선순위에 반영한다.

지속 교전의 목적은 완성된 방어를 주장하는 데 있지 않다. 방어의 사각을 우선적이고 반복적으로 식별해 배포 우선순위로 전환하는 데 있다. 본 보고서가 제시한 폐루프 구조는 이러한 반복을 감사 가능하게 하는 기반이다.
